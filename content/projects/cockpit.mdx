## Problem.

Modern car dashboards are cluttered with competing UI patterns inherited from physical gauges. Drivers are overwhelmed by information density at exactly the moments when cognitive load needs to be lowest.

## Solution.

<ProjectTabs>
  <ProjectTab label="Layout.">
    Speed and navigation in the primary zone (center), secondary data (battery, range, ADAS) in a peripheral band. The band fades to low opacity during active driving and returns at stops.

  </ProjectTab>
  <ProjectTab label="Mode.">
    The dashboard reconfigures based on driving mode: Urban, Highway, Parking. Each mode surfaces only the data relevant to that context and hides everything else.
  </ProjectTab>
  <ProjectTab label="Final.">
    Hybrid of V1 and V2. The zonal layout persists, but the peripheral band content changes with context. Tested best for cognitive load across all driving scenarios.
  </ProjectTab>
</ProjectTabs>

## Technical details.

<ProjectAccordion>
  <ProjectAccordionItem title="Pipeline.">
    The interface runs at 60fps on a custom SoC using a Vulkan-based compositor. UI layers are composited separately from the map tile renderer to avoid frame drops during navigation updates.
  </ProjectAccordionItem>
  <ProjectAccordionItem title="Typography.">
    All type is set in a custom version of DIN Next optimised for legibility at 80km/h. Minimum size 20sp. No decorative weights.
  </ProjectAccordionItem>
  <ProjectAccordionItem title="Modes.">
    Automatic switching via ambient light sensor with a 300ms crossfade. Night mode shifts the primary hue from white to amber to reduce blue light scatter on the windshield.
  </ProjectAccordionItem>
</ProjectAccordion>
