Kodomi

[koh-doh-mee]

A no-code platform designed to create websites with maximum flexibility in customization. The platform integrates UI libraries via GitHub, allowing users to flexibly customize the design and functionality of the site. Includes a marketplace for buying and selling UI components, and also provides a powerful tool for working with visual elements via AI Designer and a unique sitemap editor. These features allow users to create professional websites without the need for programming.

Role

As a lead designer, I was responsible for designing the user experience, developing the user flow, creating wireframes and a token-based design system.

Contribution

I designed key interfaces, ran multiple iterations and tests, analyzed feedback, and made decisions on navigation, customization of elements, and the content management system.

Controls

To create a flexible editing system, it was important to maintain a balance between functionality and usability. I developed customizable drag-and-drop controllers and implemented disclosures to change the data type. This allowed users to flexibly customize UI components without overloading the interface.

Research

User research and prototype testing helped identify key improvements and refine the experience for different user types. Initial testing included an analysis of the user path: from the first point of contact with the product to interaction with key features, such as configuring UI components, working with the Sitemap Editor, and using the AI ​​Designer. The insights gained influenced the further product strategy and development of the platform.

Layers and pages

The focus was on how to best organize the display of layers and pages, as well as their arrangement for the convenience of users. As a designer, I made key decisions to improve the visibility of nested layers and their states. To improve the clarity of the interface, all elements were drawn in components using boolean and text options, which made it easier to navigate and interact with the interface, improving the user experience and reducing the likelihood of errors.

Testing

The research began with self-auditing, competitive analysis, collecting and analyzing metrics, and creating user personas. In the next step, I formed hypotheses and testing scenarios, developing structured interview scripts to ensure consistency. The tests involved designers and developers with different levels of experience, which helped to understand how adaptable the platform was to different categories of users. Engineers and product managers were also involved in the interviews to ensure transparency of the process and accuracy of the interpretation of the data.

Comparison

To evaluate the effectiveness of Kodomi, I conducted tests comparing workflows with similar platforms - Webflow, Wix, and Figma. Participants performed the same tasks in Webflow and Kodomi, including creating a page structure and customizing UI elements. Testing confirmed that Kodomi's Sitemap Editor significantly speeds up page management, making site structure more intuitive than in Webflow.

Challenge 1

Kodomi needed to provide extensive UI customization options while remaining user-friendly. The challenge was to integrate advanced design controls without making the interface overly complex or overwhelming for users with different experience levels.

Solution

I designed a drag-and-drop control system, allowing users to reposition and modify UI components effortlessly. Additionally, I introduced disclosures for input fields, enabling seamless selection and modification of data types while maintaining a clean and intuitive interface.

Challenge 2

Integrating third-party UI components can lead to inconsistent designs, making it challenging to maintain a cohesive user experience across different elements. Without a structured approach, users might end up with visually disconnected components.

Solution

I introduced assets editor ensuring that all UI elements adhere to a consistent design language while allowing deep customization. This provided users with both flexibility and a structured design system.

Assets Editor

To effectively manage the design, I developed the Assets Editor, which allows users to view, edit, and organize UI components. This made it possible to quickly change elements while maintaining the overall style of the project.

Challenge 3

Traditional website builders require users to manually assemble components, slowing down the design process. Users needed a faster way to build websites without sacrificing creative control.

Solution

I implemented AI Designer, which suggests and applies UI components automatically based on user selections. This system can generate full layouts or refine existing designs, cutting down customization time significantly while still allowing manual adjustments.

AI Designer

By leveraging intelligent algorithms, I ensured that users could quickly generate custom designs for elements or entire pages based on their preferences, simplifying the design process. I focused on creating a system where AI could adapt existing designs or apply changes, all while maintaining the project's visual unity. Additionally, I integrated UI bundles, allowing the AI to automatically select and fit components seamlessly, enhancing user creativity while ensuring consistency and professional results.

Marketplace

The internal marketplace allows users to find, buy, or download UI elements created by the community. This allows users to extend the functionality of their sites without complex setup.

Navigation

An important aspect was improving the navigation bar in the editing mode, where my main goal was to reduce the number of transitions between pages, while maintaining the functionality of each page and their usability. As a designer, I made decisions aimed at ensuring easy access to key functions such as publishing, sharing, commenting and previewing, which improved the overall user experience and made working with the project more intuitive and efficient.

Elements

Kodomi allows you to use ready-made UI elements that can be easily integrated into a project. This lowers the entry threshold for users without coding experience and speeds up the process of creating websites. The platform supports connecting UI components directly through GitHub. This allows you to use third-party libraries, as well as connect your own prebuilt elements, managing their versions and updates. All prebuilt elements support customization through tokens, allowing you to change colors, sizes, and styles to suit the needs of the project. Users can also edit downloaded elements and adapt them to their tasks.

Challenge 4

Website builders often struggle with scalable page management, making it difficult for users to maintain a structured and organized project. Many competitors lacked a visual site structure, leading to confusion when handling multiple pages.

Solution

I created Sitemap Editor, which provides a visual representation of the entire site structure. Users can not only navigate pages easily but also define page type inheritance, ensuring consistency across designs and significantly reducing the time spent on managing layouts.

Sitemap Editor

One of the key features was the Sitemap Editor, which competitors do not have. I decided to integrate it for visual management of pages and the links between them. Users can not only view the structure, but also set page types, inherit the design and edit properties, which significantly speeds up work with the site.

Feedback

After testing the interactive prototype, I collected feedback from respondents. Three quarters of users noted the convenience of the Sitemap Editor, which simplified page management, and the AI Designer accelerated the creation of the UI compared to manual configuration. The collected comments were analyzed and used to improve the interface, which became the basis for the second iteration of the design.

Efficiency

The results of the tests showed that creating the basic structure of the site in Kodomi was faster than in Webflow, due to the more convenient organization of pages. Customizing UI components was also more efficient using AI Designer, which significantly reduced the time of work. These data confirmed the value of AI tools for process optimization and became the basis for further development of this feature.

Learned

Working on Kodomi, I delved into designing complex no-code systems and creating flexible UI solutions. One of the main takeaways was the importance of balancing functionality and simplicity of the interface. Building a sitemap editor, customization system, and AI tools showed that users value flexibility but don't want a cluttered experience. I also improved my skills in usability testing and data analysis, which helped me make more effective user-focused decision making.

Conclusion

Kodomi became the foundation for further development of the idea of ​​a no-code web platform with deep customization capabilities. The developed design-system approach and integration of UI libraries through GitHub created the basis for scaling the product. The insights gained will help me in future projects, especially in developing complex tools focused on flexibility, convenience, and integration with existing ecosystems.

Next

Next steps for Kodomi involve further development of the marketplace for buying and selling assets, expanding customization settings, and refining the overall project functionality. This includes enhancing the AI Designer, improving the sitemap editor, integrating more third-party UI libraries, and preparing the platform for market entry. We will also focus on user testing and securing partnerships to scale Kodomi's reach and capabilities.
